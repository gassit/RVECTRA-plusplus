# Цифровой двойник электросети - Контекст проекта

## Описание проекта
Автономное веб-приложение для моделирования электрической сети без внешних API и ИИ. Все расчёты выполняются локально.

## Расположение проекта
```
/home/z/my-project/network-digital-twin/
```

## Технологии
- **Frontend**: Next.js 16, React, TypeScript, Tailwind CSS 4
- **Граф сети**: Cytoscape.js
- **База данных**: Prisma ORM + SQLite (LibSQL)
- **UI**: shadcn/ui компоненты

## Структура базы данных (19 таблиц)
```
prisma/schema.prisma
```

### Основные таблицы:
- `Element` - узлы сети (источники, шины, автоматы, нагрузки и т.д.)
- `Connection` - связи между элементами
- `Cable` - кабели с параметрами
- `Device/DeviceSlot` - устройства и слоты
- `Breaker/Meter/Transformer/Load` - типы устройств
- `CableReference/BreakerReference/TransformerReference` - справочники по ПУЭ
- `ValidationRule/ValidationResult` - правила и результаты валидации
- `CalculatedParams` - расчётные параметры
- `Alarm` - аварийные сигналы

## Цвета узлов
| Тип | Цвет | Форма |
|-----|------|-------|
| source | Градиент жёлтый→зелёный→красный | Октагон |
| breaker | Белый | RoundRectangle |
| load | Чёрный | Rectangle |
| meter | Синий | Diamond |
| bus | Медный (amber) | Удлинённый прямоугольник |
| junction | Серый | Ellipse |

## Правила размещения узлов
1. **source** - вверху схемы
2. **load** - внизу схемы
3. Элементы "to" от Bus - на одном уровне под Bus
4. Элементы "from" питающие Bus - над Bus
5. load под breaker при связи breaker→load

## Правила валидации защиты
1. **Согласование автомат-кабель**: Iном.выкл ≤ Iдоп.кабеля
2. **Потеря напряжения**: ΔU ≤ 4%
3. **Ток КЗ в конце линии**: Iкз ≥ 3×Iном.автомата
4. **Селективность**: Iном.верх ≥ Iном.низ
